Home Field Advantage · Natural Experiment

The 2020 Effect

Home field advantage is baked into every spread. The 2020 COVID season — zero fans in stadiums — created the cleanest natural experiment in sports analytics history. It reveals exactly how much of "home field" is crowd noise versus genuine logistical advantage.

Methodology · Data · Limitations

How It Works

This dashboard loads and processes 25 years of real NFL data live in your browser — no pre-computed numbers, no server. Everything is calculated fresh from the raw CSV on every page load. Here's exactly what's being computed and where the data comes from.

Technical Approach
◈
DATA SOURCE
nflverse/nfldata — open-source NFL data maintained by the analytics community and used by league teams. Spread lines are consensus closing lines from major sportsbooks. Raw CSV fetched live via fetch() on page load using PapaParse.
Δ
ATS CALCULATION
Home covered if: (home_score − away_score) + spread_line > 0. Push if exactly 0. spread_line is negative when home is favored, positive when home is underdog. Push games excluded from cover rate denominator.
∑
TOTALS CALCULATION
Over hits if combined score exceeds total_line. Over rate = overs / (overs + unders), push games excluded. Dome classification uses the roof field: dome, outdoors, retractable, open.
⚠
KEY LIMITATIONS
Dataset contains one spread_line per game (closing consensus) — no opening line and no line movement data. Ticket% / money% data is proprietary to sportsbooks and has no public historical source. Early seasons (1999–2003) have occasional missing moneyline data.
σ
STATISTICAL NOTES
With ~6,500 games a 1pp deviation from 50% ATS represents ~65 games — meaningful signal. Team-level charts require ≥150 games to filter small-sample noise. The 2020 COVID season is the strongest natural experiment in sports analytics for measuring crowd effect.
↻
LIVE & SELF-UPDATING
As nflverse pushes new games, this dashboard updates automatically on the next page load — no rebuild or redeployment required. PapaParse streams the CSV for efficient in-browser processing of 6,500+ rows.
